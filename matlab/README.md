# 🦾 Robotic Arm — MATLAB Simscape Model

A full robotic arm simulation built with MATLAB & Simscape Multibody, covering kinematics, trajectory planning, and servo control.

---

## 📁 Project Structure

```
assembly/
├── Assem2.slx                        # Main Simscape Multibody model
├── Assem2_DataFile.m                 # Auto-generated model data (rigid transforms, parameters)
├── rigidtree.m                       # Imports robot model & builds rigid body tree
│
├── Assem2_fkin.slx                   # Forward kinematics simulation
├── Assem2_inversekin.slx             # Inverse kinematics simulation
├── Assem2_jacobian.slx               # Jacobian matrix computation
│
├── Assem2_trajectoryplanning.slx     # Trajectory planning (end-effector path)
├── Assem2_trajectoryangles.slx       # Trajectory planning (joint angles output)
│
├── Assem2_p2p.slx                    # Point-to-point motion test
├── Assem2_p2pangles.slx              # Point-to-point motion (joint angles output)
│
├── trajec.m                          # Trajectory waypoints, matrices & shared workspace data
├── servo_control.slx                 # Servo control — manual slider interface
└── trajectory_servo_control.slx      # Servo control — runs angles from trajec.m
```

---

## ▶️ How to Run

> **Order matters.** Always follow the steps below — skipping a step will cause errors.

### Step 1 — Initialize the Model Data

Run `Assem2_DataFile.m` **first**, every time you open a new MATLAB session. This loads all rigid body transform data (translations, rotations, joint IDs) required by the Simscape model.

```matlab
run('Assem2_DataFile.m')
```

> This file is auto-generated by Simscape Multibody Import and should **not** be manually edited beyond numerical values.

---

### Step 2 — Build the Rigid Body Tree

Before running any kinematics or Jacobian scripts, run `rigidtree.m` to import the robot and construct its rigid body tree representation.

```matlab
[dof, ArmInfo] = importrobot('Assem2.slx')
% or simply:
run('rigidtree.m')
```

This gives you:
- `dof` — degrees of freedom of the robot
- `ArmInfo` — full rigid body tree used in all downstream computations

---

### Step 3A — Kinematics & Jacobian

Once Step 1 and Step 2 are complete, you can open and run any of the following Simulink models independently:

| File | Description |
|---|---|
| `Assem2_fkin.slx` | Computes end-effector pose from joint angles |
| `Assem2_inversekin.slx` | Solves joint angles for a target end-effector pose |
| `Assem2_jacobian.slx` | Computes the Jacobian matrix at a given configuration |

---

### Step 3B — Trajectory Planning & Servo Control

For trajectory planning or servo control, an **additional setup step** is required. Run `trajec.m` first to load the waypoints, transformation matrices, and shared variables needed by these models.

```matlab
run('trajec.m')
```

Then open and simulate any of the following:

| File | Description |
|---|---|
| `Assem2_trajectoryplanning.slx` | Trajectory planning — end-effector Cartesian path |
| `Assem2_trajectoryangles.slx` | Trajectory planning — joint angles output |
| `Assem2_p2p.slx` | Point-to-point motion test |
| `Assem2_p2pangles.slx` | Point-to-point motion — joint angles output |
| `trajectory_servo_control.slx` | Servo control driven by angles from `trajec.m` |

The `servo_control.slx` model does **not** require `trajec.m` — it uses built-in sliders for manual joint control:

| File | Description |
|---|---|
| `servo_control.slx` | Manual servo control via interactive sliders |

```
1. Assem2_DataFile.m                    ← Always first

   + trajec.m                          ← Also required for trajectory & automated control
```

## 📌 Notes

- Do **not** edit the physical units in `Assem2_DataFile.m` comments — only numerical values may be changed.
- Do **not** add arbitrary code to `Assem2_DataFile.m`.
- All scripts assume the current MATLAB working directory is set to the `assembly/` folder.
